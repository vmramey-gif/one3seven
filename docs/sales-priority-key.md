# Sales Priority Key — how to work the list

**Work it top to bottom. Call order is by TIER. The pitch changes by tier — it's pre-written in each firm's Call brief.**

---

## TIER = call order + which pitch to use (the main signal)
Open a firm in the CRM → the **Call brief** shows its tier badge, the **Angle** to lead with, and the **objection prep**.

| Tier | Who | Call order | Lead with |
|---|---|---|---|
| 🟢 **Tier 1 — Tech-native** (10) | Already run AI in their firm (e.g. D.Law, Bohm) | **Call first** | Skip the AI pitch — "you already speak AI; we close the *intake* gap." Objections are price/fit/timing, **not** AI skepticism. |
| 🔵 **Tier 2 — Growth** (84) | Multi-attorney / high-volume, feel intake pain at scale | **Best ROI/hour — call second** | The staffing bottleneck: "it's not caseload, it's staff time sorting intake — we eliminate that step." |
| 🟡 **Tier 3 — Modern boutique** (217) | Solo/boutique, modern enough to be receptive | **Call third** | Quality triage: "see the full picture before the first consult, decide faster." |
| ⚪ **Tier 4 — Traditional** (39) | Longer cycle, pipeline depth | **Call last** | Simplicity: "worker uploads records, we organize it, you get a clean packet. No jargon. Free pilot." Don't oversell tech — sell the packet. |

- **PRIORITY TARGET / FOUNDER** → first calls; warm problem statement, not a cold call.
- **WARM REGION** → Central Valley/near-Tracy; open with the local connection.

## The one-line sequence for a rep
1. **A-priority firms first** (5) → then
2. **Tier 1** (10) → **Tier 2** (84) → **Tier 3** (217) → **Tier 4** (39).
3. For each: open the firm → read the **Call brief** (tier angle + objection prep) → tap-to-call → fast-log the outcome.
4. First rep to log **claims** the firm (it drops off the shared list). Set a next-follow-up date on every call.
