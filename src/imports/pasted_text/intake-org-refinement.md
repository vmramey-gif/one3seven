Refine the One3Seven intake organization logic so that all worker-provided information is reviewed together as part of one connected intake workspace.

IMPORTANT:
Do not redesign the app.
Do not change the existing visual identity, layout, or workflow structure.
This prompt should refine the intake organization behavior, AI review logic, saved context behavior, filename handling, and intake summary quality.

CORE ORGANIZATION LOGIC

The intake organization process should consider ALL worker-provided information together, including:

* uploaded files
* scanned or extracted file contents
* original file names
* worker-edited file names
* document categories
* context added inside timeline cards
* notes/context entered during intake
* additional information shared by the worker
* renamed files and user-provided file descriptions

The app should treat all of this as connected intake information.

File names and renamed file names should help inform:

* where the document belongs
* what type of record it may be
* which timeline event it may relate to
* which document category it should be grouped under
* how it should be referenced in the intake summary

If a worker renames a file, that edited filename should be considered meaningful user-provided context and should be used when organizing the intake.

TIMELINE ADD DETAILS BEHAVIOR

When a worker opens an Employment Timeline card and enters text in the Add Details / Additional Context field:

* the typed context should save directly inside that same timeline card
* each timeline card should preserve its own independent saved context
* saved timeline context should reappear when the same card is reopened
* saved timeline context should not disappear, reset, or attach to the wrong timeline card

The saved context should also be reviewed by the AI organization logic and used to enhance:

* timeline continuity
* factual chronology
* organization summaries
* intake summary quality
* document-to-event connections
* context around why certain records matter

The context should help improve organization and factual clarity only.

Do NOT use worker-added context to:

* make legal conclusions
* score claim strength
* predict case outcomes
* determine legal merit

DOCUMENT SORTING + CONTEXT CONNECTION

The AI organization logic should sort and connect records by considering:

* file contents
* file names
* edited file names
* extracted dates
* document type
* worker notes
* worker-provided context
* related timeline events
* repeated issue patterns
* supporting records

Documents should be grouped in a way that helps tell the factual story the worker is trying to convey.

The organization should help participating firms quickly understand:

* what happened
* when it happened
* what records support it
* what the worker added for context
* where additional documents may be helpful

The organization should feel like:
“scattered records and worker context becoming a coherent intake story.”

INTAKE SUMMARY QUALITY

The Intake Summary should NOT feel like short, generic bullet points.

Avoid:

* vague summaries
* generic AI language
* shallow bullet lists
* robotic phrasing
* overly short summaries that miss context
* legal conclusions or claim-strength statements

Instead, generate a richer AI-organized factual intake narrative.

The Intake Summary should feel like a structured factual story built from:

* uploaded documents
* extracted file information
* file names and renamed files
* timeline context
* worker notes
* added details
* document categories
* chronology
* supporting evidence references

The summary should help a participating law firm understand the matter quickly and feel interested in reviewing the organized intake.

SUMMARY STYLE

The intake summary should be:

* factual
* organized
* readable
* professional
* narrative-driven
* intake-focused
* emotionally grounded without being dramatic
* clear enough for law firm review
* detailed enough to avoid feeling generic

The summary should include sections such as:

1. Intake Overview
   A clear narrative overview of the worker’s situation based on uploaded documents and provided context.

2. Chronology of Events
   A structured timeline narrative showing the sequence of events and how records connect.

3. Worker-Provided Context
   Preserve meaningful worker-provided context without rewriting everything into robotic AI language.

4. Supporting Records
   Explain which document categories support which parts of the chronology.

5. Organization Notes
   Identify workflow-relevant observations such as incomplete date ranges, missing supporting records, or document clusters.

Use careful language such as:

* “records indicate”
* “uploaded materials suggest”
* “worker-provided context states”
* “additional records may help clarify”
* “timeline continuity may be strengthened by”

Avoid language such as:

* “this proves”
* “the employer violated”
* “strong case”
* “weak case”
* “legal claim confirmed”
* “likely outcome”
* “merit score”

DOCUMENT INVENTORY + INTAKE SUMMARY CONNECTION

The Document Inventory should connect meaningfully to the Intake Summary.

When possible:

* document groups should reference related timeline events
* timeline events should reference related documents
* worker-added context should appear connected to the relevant chronology card
* renamed files should appear using the worker-edited filename
* supporting records should be described in plain, factual language

FINAL GOAL

The final organized output should feel like:

“One3Seven reviewed the uploaded records, filenames, worker notes, and added context together, then organized them into a coherent factual intake story that a participating law firm can understand quickly.”

Maintain One3Seven’s positioning as an intake organization and workflow preparation platform only.

One3Seven does not:

* provide legal advice
* determine legal claim validity
* recommend legal outcomes
* score case strength
* establish attorney-client relationships

The platform organizes information for intake preparation and participating-firm review.
